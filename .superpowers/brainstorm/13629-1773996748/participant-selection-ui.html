How should participants select lines?

Participants see a read-only code snippet and click lines to flag them

A

Gutter Click (like an IDE)

Clicking the line number area (left gutter) toggles selection. The line highlights. Clicking the code itself does nothing — prevents accidental selections. Familiar to developers.

1 public String process(String input) {
2 ● String result = input.trim().toLowerCase();
3 if (result.isEmpty()) {
4 ● return null;
5 }
6 return result + result;

● appears in gutter for selected lines. Click again to deselect.

B

Full Row Click

Clicking anywhere on a line toggles selection. Simpler, more forgiving touch targets — works better on mobile. Entire row lights up.

1 public String process(String input) {
2 String result = input.trim().toLowerCase(); ✓
3 if (result.isEmpty()) {
4 return null; ✓
5 }

Tap/click anywhere on the row. ✓ appears on selected lines.
