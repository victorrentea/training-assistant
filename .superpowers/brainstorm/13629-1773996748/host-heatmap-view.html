How should the host see participant line selections?

The host needs to see which lines participants are clicking, in real-time

A

Heatmap Overlay

Each line gets a colored background whose intensity reflects how many participants selected it. Darker = more selections. The host sees the code with a gradient overlay — instant visual signal of "hot" lines.

2 String result = input.trim().toLowerCase(); ● 18 selections

4 return null; ● 27 selections
5 }
6 return result + result; ● 9 selections
7 }

Host clicks a line → expands participant list sorted by score (ascending)

B

Heatmap + Side Panel

Same heatmap overlay, but clicking a line opens a side panel showing the full list of participants who selected it. Two-pane layout: code left, participant list right.

C

Inline Expansion

No side panel. Clicking a hot line expands it inline to reveal the participant list directly below that line in the code view. Compact, no layout shift.
