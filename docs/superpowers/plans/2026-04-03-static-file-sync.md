# Static File Sync — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable the daemon to upload static files (HTML/JS/CSS) to Railway on connect, so static-only changes deploy instantly without a Railway rebuild (~40-50s savings).

**Architecture:** On daemon WS connect, backend sends a `sync_files` message containing MD5 hashes of all files in `static/`. Daemon diffs against its local `static/` folder and uploads any changed files via HTTP POST to `/internal/upload-static`. After uploading, daemon broadcasts a `reload` event so participant browsers refresh. This supplements (does not replace) the existing Railway auto-deploy for backend changes.

**Tech Stack:** Python 3.12, FastAPI, `websockets` library (daemon-side), pytest

**GitHub Issue:** Part of railway proxy redesign (#77)

---

## Context

Currently, **every** change — even a CSS tweak — requires `git push` → Railway rebuild (40-50s). The daemon already has a persistent WS connection to the backend. This plan adds a fast-path: daemon detects local static file changes and uploads them directly, bypassing Railway's build pipeline for static-only changes.

This is **Phase 1** of the railway proxy redesign spec (`docs/railway-proxy-redesign.md`).

### What changes vs what stays

- **Changes:** Daemon uploads static files on connect; backend has `/internal/*` HTTP endpoints for file management; `sync_files` WS message on daemon connect
- **Stays:** Railway auto-deploy still works for backend changes; `version.js` still generated at Railway startup; `deploy-info.json` still generated by GitHub Actions; `version-reload.js` auto-reload still works; `watch-deploy.sh` still works

### Current deploy flow (for reference)

```
git push → Railway rebuild (40-50s) → version.js stamped → version-reload.js detects change → browsers auto-reload
```

### New fast-path (added by this plan)

```
Daemon starts/reconnects → compares static/ hashes → uploads changed files via HTTP → broadcasts "reload" → browsers refresh
```

## Design Decisions

### Auth for `/internal/*` endpoints
Reuse existing Basic Auth (HOST_USERNAME/HOST_PASSWORD). Same credentials as daemon WS. No new secrets. **Note:** The spec says `/internal/*` has "no auth" — we deviate intentionally since these endpoints are exposed over the public internet.

### Hash algorithm
MD5 of file contents. Fast, sufficient for change detection (not security-critical).

### File inventory scope
Recursive scan of `static/` directory (includes `static/avatars/`). Paths are relative to `static/` (e.g., `avatars/gandalf.png`). Excludes `version.js` (generated at Railway startup), `deploy-info.json` (generated by GitHub Actions / Railway startup), and `work-hours.js` (manually maintained per CLAUDE.md).

### Upload size limit
Max 5MB per file (decoded). Static files are typically 50-200KB; this prevents accidental or malicious oversized uploads.

### Forward-compatible `sync_files` message
The `sync_files` message includes both `static_hashes` and `pdf_slugs` (empty `{}` for now). This matches the spec and avoids a protocol change when Phase 2 PDF sync is added.

### Upload encoding
Base64-encoded file content in JSON body (files are small: ~50-200KB each). No multipart needed.

### Reload trigger
Daemon sends `{"type": "broadcast_event", "event": {"type": "reload"}}` via WS after all uploads complete. Backend fans out `{"type": "reload"}` to all participant WS connections. `version-reload.js` already handles page reloads — we'll add a `reload` event listener alongside the existing version-check logic.

### Daemon-side file watching
**Not included in this plan.** Daemon only syncs on WS connect (startup + reconnect). Future enhancement: watch `static/` for changes and auto-upload. For now, restart daemon to trigger re-sync.

### Excluded files
- `version.js` — generated at Railway startup, not synced from daemon
- `deploy-info.json` — generated by GHA/Railway, not synced from daemon
- `work-hours.js` — per CLAUDE.md, never auto-edited

All three must be in both `_EXCLUDED_FILES` (backend router) and `_EXCLUDED` (daemon sync module).

# features/internal/router.py
"""Internal endpoints for daemon → backend file management."""
import base64
import logging
import os
from pathlib import Path

from fastapi import APIRouter, Depends, HTTPException
from pydantic import BaseModel

from core.auth import require_host_auth

router = APIRouter(prefix="/internal", dependencies=[Depends(require_host_auth)])
logger = logging.getLogger(__name__)

_STATIC_DIR = Path(__file__).resolve().parent.parent.parent / "static"

# Only allow uploading to static/ — no path traversal
_ALLOWED_EXTENSIONS = {".html", ".js", ".css", ".png", ".jpg", ".svg", ".ico", ".json", ".woff", ".woff2", ".ttf"}
_EXCLUDED_FILES = {"version.js", "deploy-info.json", "work-hours.js"}
_MAX_UPLOAD_SIZE = 5 * 1024 * 1024  # 5MB decoded


class UploadStaticRequest(BaseModel):
    path: str       # relative to static/, e.g. "participant.js"
    content_b64: str  # base64-encoded file content


class DeleteStaticRequest(BaseModel):
    path: str       # relative to static/


def _validate_path(rel_path: str) -> Path:
    """Validate and resolve a relative path within static/. Raises HTTPException on violation."""
    clean = rel_path.strip().replace("\\", "/")
    if ".." in clean or clean.startswith("/"):
        raise HTTPException(status_code=400, detail="Invalid path")
    resolved = (_STATIC_DIR / clean).resolve()
    if not str(resolved).startswith(str(_STATIC_DIR.resolve())):
        raise HTTPException(status_code=400, detail="Path traversal denied")
    if resolved.name in _EXCLUDED_FILES:
        raise HTTPException(status_code=400, detail=f"Cannot overwrite {resolved.name}")
    suffix = resolved.suffix.lower()
    if suffix not in _ALLOWED_EXTENSIONS:
        raise HTTPException(status_code=400, detail=f"Extension {suffix} not allowed")
    return resolved


@router.post("/upload-static")
async def upload_static(req: UploadStaticRequest):
    """Upload or overwrite a static file. Called by daemon during static file sync."""
    target = _validate_path(req.path)
    try:
        content = base64.b64decode(req.content_b64)
    except Exception:
        raise HTTPException(status_code=400, detail="Invalid base64 content")
    if len(content) > _MAX_UPLOAD_SIZE:
        raise HTTPException(status_code=400, detail=f"File too large ({len(content)} bytes, max {_MAX_UPLOAD_SIZE})")
    target.parent.mkdir(parents=True, exist_ok=True)
    target.write_bytes(content)
    logger.info("Static file uploaded: %s (%d bytes)", req.path, len(content))
    return {"status": "ok", "path": req.path, "size": len(content)}


@router.post("/delete-static")
async def delete_static(req: DeleteStaticRequest):
    """Delete a static file. Called by daemon during static file sync."""
    target = _validate_path(req.path)
    if not target.exists():
        return {"status": "ok", "action": "not_found"}
    target.unlink()
    logger.info("Static file deleted: %s", req.path)
    return {"status": "ok", "action": "deleted"}
```

# daemon/static_sync.py
"""Static file sync — compare local static/ with backend inventory and upload changes."""
import base64
import hashlib
from pathlib import Path

from daemon import log
from daemon.http import _post_json

_EXCLUDED = {"version.js", "deploy-info.json", "work-hours.js"}


def compute_local_hashes(static_dir: Path) -> dict[str, str]:
    """Build {relative_path: md5_hex} for all files in a static directory (recursive)."""
    hashes = {}
    if static_dir.is_dir():
        for f in static_dir.rglob("*"):
            if f.is_file() and f.name not in _EXCLUDED:
                rel = str(f.relative_to(static_dir))
                md5 = hashlib.md5(f.read_bytes()).hexdigest()
                hashes[rel] = md5
    return hashes


def diff_hashes(local: dict[str, str], remote: dict[str, str]) -> tuple[list[str], list[str]]:
    """Compare local and remote hashes.

    Returns:
        (to_upload, to_delete): filenames that need uploading or deleting on the remote.
    """
    to_upload = []
    for name, local_hash in local.items():
        if name not in remote or remote[name] != local_hash:
            to_upload.append(name)

    to_delete = []
    for name in remote:
        if name not in local:
            to_delete.append(name)

    return to_upload, to_delete


def sync_static_files(
    static_dir: Path,
    remote_hashes: dict[str, str],
    server_url: str,
    username: str,
    password: str,
) -> int:
    """Sync local static/ to backend. Returns number of files changed."""
    local_hashes = compute_local_hashes(static_dir)
    to_upload, to_delete = diff_hashes(local_hashes, remote_hashes)

    if not to_upload and not to_delete:
        log.info("static-sync", "Static files in sync — no changes")
        return 0

    changed = 0

    for name in to_upload:
        filepath = static_dir / name
        if not filepath.exists():
            continue
        content_b64 = base64.b64encode(filepath.read_bytes()).decode()
        try:
            _post_json(
                f"{server_url}/internal/upload-static",
                {"path": name, "content_b64": content_b64},
                username, password,
            )
            log.info("static-sync", f"Uploaded: {name} ({filepath.stat().st_size} bytes)")
            changed += 1
        except RuntimeError as e:
            log.error("static-sync", f"Failed to upload {name}: {e}")

    for name in to_delete:
        try:
            _post_json(
                f"{server_url}/internal/delete-static",
                {"path": name},
                username, password,
            )
            log.info("static-sync", f"Deleted remote: {name}")
            changed += 1
        except RuntimeError as e:
            log.error("static-sync", f"Failed to delete {name}: {e}")

    return changed
```

"""Tests for daemon static file sync logic."""
import hashlib
from pathlib import Path

from daemon.static_sync import compute_local_hashes, diff_hashes, sync_static_files


class TestComputeLocalHashes:
    def test_hashes_files(self, tmp_path):
        (tmp_path / "a.js").write_text("console.log('a');")
        (tmp_path / "b.css").write_text("body { color: red; }")
        hashes = compute_local_hashes(tmp_path)
        assert "a.js" in hashes
        assert "b.css" in hashes

        (tmp_path / "version.js").write_text("window.APP_VERSION='test';")
        (tmp_path / "app.js").write_text("ok")
        hashes = compute_local_hashes(tmp_path)
        assert "version.js" not in hashes
        assert "app.js" in hashes

    def test_excludes_deploy_info(self, tmp_path):
        (tmp_path / "deploy-info.json").write_text("{}")
        hashes = compute_local_hashes(tmp_path)
        assert "deploy-info.json" not in hashes

class TestDiffHashes:

        assert to_upload == ["a.js"]
        assert to_delete == []

    def test_changed_file(self):

        assert to_upload == ["a.js"]
        assert to_delete == []
